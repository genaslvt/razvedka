# 10. Документирование изменений

**Чанжлог (`CHANGELOG.md`):**

- Формат: Keep a Changelog
- Обязателен для каждого релиза

**Пользовательская документация:**

- Обновляется при изменении логики UI
- Версионируется (совпадает с релизом)

**API-документация:**

- Swagger / OpenAPI (автогенерация из кода)
- Доступна по `/api/docs`

**Хранилище документации:** Confluence / Git Wiki

**Согласование:** заказчик подписывает изменения перед релизом
